Reports invocations of input class constructors without using named parameters.

It is recommended to use name parameters or builders when creating input objects to avoid issues when the schema changes.

For example, given the following schema:

input PersonInput {
  firstName: String
  lastName: String
}
And creating the input object without using named parameters like this:
PersonInput("John", "Doe")
If the schema changes to:
input PersonInput {
  lastName: String
  firstName: String
}
The input object will be created with the wrong values, without any warning. Using named parameters or builders will avoid this issue:
PersonInput(firstName = "John", lastName = "Doe")
Or:
PersonInput.builder()
  .firstName("John")
  .lastName("Doe")
  .build()
